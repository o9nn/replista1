
# User Training Guide

## Training Program Structure

### Beginner Level (Week 1-2)

#### Day 1: Introduction
- Platform tour
- Basic chat interface
- Sending messages
- Understanding responses

**Hands-on Exercise:**
```
1. Ask: "What is JavaScript?"
2. Upload a simple .js file
3. Ask: "@filename.js Explain what this code does"
```

#### Day 2: File Management
- Uploading files
- Using @ mentions
- Viewing file content
- Removing files

**Practice Tasks:**
```
1. Upload 3 different file types
2. Mention each in a question
3. Remove one file
4. Upload it again
```

#### Day 3: Assistant Prompts
- Understanding prompts
- Selecting appropriate prompts
- When to switch prompts

**Training Activity:**
```
1. Use "Default" prompt: Ask a general question
2. Switch to "Teacher": Ask the same question
3. Compare the responses
4. Discuss differences
```

### Intermediate Level (Week 3-4)

#### Checkpoints & Rollback
- Creating checkpoints
- Naming conventions
- Restoring previous states
- Best practices

**Exercise:**
```
1. Create checkpoint "Starting point"
2. Make some code changes
3. Create checkpoint "After changes"
4. Make bad changes
5. Restore to "After changes"
```

#### Code Review Workflow
- Uploading code for review
- Using Code Reviewer prompt
- Understanding suggestions
- Implementing fixes

**Practice:**
```
1. Upload code with known issues
2. Ask for review
3. Understand each suggestion
4. Fix issues one by one
```

#### Debugging Techniques
- Providing error messages
- Sharing relevant context
- Systematic problem-solving

**Debug Exercise:**
```
1. Share error: "[Error message]"
2. Provide context: "@file.js line 42"
3. Follow debugging steps
4. Verify solution
```

### Advanced Level (Week 5-6)

#### Multi-File Projects
- Managing complex codebases
- Cross-file references
- Architecture discussions

**Advanced Task:**
```
1. Upload entire project
2. Ask architectural questions
3. Request refactoring across files
4. Implement suggestions
```

#### Optimization & Best Practices
- Performance optimization
- Code quality improvement
- Security considerations

**Workshop:**
```
1. Review existing code
2. Identify bottlenecks
3. Ask for optimization suggestions
4. Benchmark improvements
```

## Common Training Challenges

### Challenge 1: Over-reliance on AI

**Problem:** Users don't understand the code they're using

**Solution:**
- Emphasize understanding over speed
- Require explanation of AI suggestions
- Use "Teacher" prompt more often
- Code review exercises

### Challenge 2: Vague Questions

**Problem:** Users ask unclear questions and get poor results

**Solution:**
- Template questions for beginners
- Before/after examples
- Practice effective prompting
- Feedback on question quality

### Challenge 3: Ignoring Context

**Problem:** Users don't provide enough file context

**Solution:**
- Demonstrate with/without file mentions
- Show quality difference
- Emphasize @ symbol usage
- Context checklist

### Challenge 4: No Testing

**Problem:** Users apply code without testing

**Solution:**
- Mandatory testing workflow
- Checkpoint before applying changes
- Test-driven examples
- Verification checklist
