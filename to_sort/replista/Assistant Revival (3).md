# Tutorial: Mastering Assistant Prompts

  

\*\*Difficulty\*\*: Intermediate

\*\*Time\*\*: 30 minutes

\*\*Prerequisites\*\*: \[Getting Started\](./getting-started.md), \[Working with Files\](./working-with-files.md)

  

## Learning Objectives

  

\- Understand all available assistant prompts

\- Select the right prompt for each task

\- Switch prompts effectively

\- Combine prompts with file references

\- Create optimal workflows

  

## Understanding Assistant Prompts

  

Assistant prompts are pre-configured personalities that change how the AI responds to your questions.

  

### Available Prompts

  

1. \*\*Default\*\* - General-purpose assistant

2. \*\*Teacher\*\* - Educational, step-by-step explanations

3. \*\*Code Reviewer\*\* - Professional code analysis

4. \*\*Debugging Assistant\*\* - Systematic problem solving

5. \*\*Architect\*\* - High-level system design

6. \*\*Optimizer\*\* - Performance and efficiency focus

  

## Part 1: Default Prompt

  

### When to Use

\- General questions

\- Quick answers

\- Exploring new topics

\- Casual conversations

  

### Exercise 1: Default Prompt Basics

  

\`\`\`

1\. Select "Default" prompt

  

2\. Ask: "What is React?"

  

3\. Observe: Concise, informative response

  

4\. Ask: "How do I create a React component?"

  

5\. Observe: Practical, straightforward answer

\`\`\`

  

### Best For

\- ✅ Quick information

\- ✅ General coding help

\- ✅ Syntax questions

\- ✅ Tool recommendations

  

### Not Ideal For

\- ❌ Deep learning (use Teacher)

\- ❌ Code review (use Code Reviewer)

\- ❌ Complex debugging (use Debugging Assistant)

  

## Part 2: Teacher Prompt

  

### When to Use

\- Learning new concepts

\- Understanding fundamentals

\- Step-by-step tutorials

\- Educational content

  

### Exercise 2: Learning with Teacher

  

\`\`\`

1\. Switch to "Teacher" prompt

  

2\. Upload a JavaScript file with async/await code

  

3\. Ask: @async-example.js "Explain how async/await works"
